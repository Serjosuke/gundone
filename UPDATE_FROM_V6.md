# Обновление с v0.6 до v0.7

v0.7 добавляет подтипы карточек и wiki-навигацию, при этом сохраняет существующие статьи, карты, изображения, регионы, метки и таймлайн.

## Что сохранить

Перед заменой исходников обязательно сохраните:

- `.env`;
- `backend/data/` — особенно `atlas.db` и `uploads/`.

На Railway ничего выгружать не нужно, если backend Volume остаётся подключённым к `/app/data`.

## Локально через Docker

1. Остановите старую версию:

```powershell
docker compose down
```

2. Замените файлы проекта на v0.7, но верните старые `.env` и `backend/data`.

3. Запустите:

```powershell
docker compose up --build
```

При первом старте backend автоматически добавит колонку `subtype` в SQLite. Ручная миграция не нужна.

## На Railway

1. Замените код в GitHub-репозитории на v0.7 и сделайте push.
2. Убедитесь, что у backend по-прежнему есть Volume с mount path `/app/data`.
3. Railway пересоберёт backend и frontend.
4. После открытия `/admin` создайте или отредактируйте карточку и заполните поле **«Подтип»**.

## Новые правила текста

- ссылка на статью: `[[Название статьи]]`;
- ссылка с другим текстом: `[[Название статьи|Текст ссылки]]`;
- тег одним словом: `#Север`;
- тег с пробелом: `[[#Главная линия]]`.
