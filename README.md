# Atlas Forge — интерактивная карта и wiki

Интерактивная world-wiki для DnD: несколько карт мира, связанные статьи, регионы, карточки на карте и закрытый редактор.

## Что уже есть

- несколько независимых карт: у каждой свой фон, набор меток, регионов и изображений;
- плавный зум колесом мыши, привязанный к месту под курсором, плюс кнопки масштаба и сброс вида;
- интерактивная карта: перетаскивание, фильтр по типу карточки;
- метки, регионы и карточки-изображения, связанные со статьями;
- одна статья может быть отмечена на нескольких картах; в статье показан список всех таких размещений с быстрым переходом на нужную карту;
- создание новой метки кликом по карте в режиме редактирования;
- перетаскивание маркеров с сохранением координат;
- карточки-статьи: заголовок, тип, описание, теги, основной текст, связи;
- создание / изменение / удаление статей;
- SQLite-персистентность: после перезапуска данные сохраняются;
- тёмный авторский дизайн без копирования vvd.world.

## Запуск через Docker

```bash
docker compose up --build
```

Откройте: `http://localhost:3000`

API: `http://localhost:8000/docs`

## Локальный запуск

### Backend

```bash
cd backend
python -m venv .venv
# Windows: .venv\\Scripts\\activate
# macOS/Linux:
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Откройте `http://localhost:3000`.

## Основные API

- `GET /api/cards` — список статей
- `POST /api/cards` — создать статью
- `PATCH /api/cards/{id}` — изменить статью
- `DELETE /api/cards/{id}` — удалить статью
- `GET /api/maps` — список карт мира
- `POST /api/maps` — создать карту
- `GET /api/maps/{map_id}` — выбранная карта и её объекты
- `DELETE /api/maps/{map_id}` — удалить карту, не затрагивая общие статьи
- `GET /api/cards/{card_id}/placements` — все привязки статьи на разных картах
- `POST /api/maps/{map_id}/markers` — добавить метку
- `PATCH /api/maps/{map_id}/markers/{id}` — переместить / изменить метку

## Управление картой в v0.4

В админке карта работает через четыре понятных режима:

- **Обзор** — ЛКМ по пустому месту двигает карту, а по метке/региону/карточке открывает объект.
- **Редактировать** — ЛКМ переносит метки и изображения. Выделенный объект можно растянуть за зелёную ручку внизу справа. Для региона становятся видны белые вершины — их тоже можно двигать ЛКМ.
- **Метка** — клик по свободному месту вызывает выбор существующей статьи или создание новой.
- **Регион** — ставьте вершины, затем нажмите «Сохранить».

У загруженных изображений теперь сохраняется собственная пропорция, поэтому и иллюстрации, и карточки меняют размер одинаково и не «прыгают».


## Новое в v0.5

### Несколько карт

В левой панели и в селекторе над картой можно переключаться между картами. В режиме хранителя нажмите **«+ Карта»**, укажите название и описание, затем загрузите отдельный фон через кнопку **«↑ Фон»**.

Метки, регионы и иллюстрации принадлежат только текущей карте. Статьи принадлежат всему миру: одна и та же статья может иметь метки на нескольких картах. При удалении карты удаляются только её объекты, но не wiki-статьи.

### Зум

Прокрутка колеса над картой приближает или отдаляет изображение именно относительно курсора. ЛКМ по свободному месту в режиме **«Обзор»** перемещает карту. Кнопка **↺** сбрасывает вид.

### Обновление с v0.4

Сохраните свои `backend/data` и `.env`, затем замените исходники. База SQLite обновляется без миграций вручную — таблица `maps` уже была в прошлой версии. После замены выполните:

```powershell
docker compose down
docker compose up --build
```
