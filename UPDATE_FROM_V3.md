# Обновление Atlas Forge с v0.3 до v0.4

## Что исправлено

- В режиме **«Обзор»** метки, регионы и карточки на карте снова кликабельны: нажатие по объекту не запускает перетаскивание карты.
- Режимы упрощены:
  - **Обзор** — открыть объект или перетащить свободную часть карты;
  - **Редактировать** — передвинуть метку/изображение, изменить размер картинки, двигать вершины региона;
  - **Метка** — поставить метку и выбрать уже существующую статью либо создать новую;
  - **Регион** — нарисовать границу.
- Все поля выбора и обычные поля используют тёмную палитру: белого текста на белом фоне больше нет.
- Изображения и карточки на карте сохраняют исходную пропорцию. Размер теперь регулируется одинаково для обоих типов: ползунком или ручкой в правом нижнем углу выделенного объекта.
- Для выделенного региона в режиме «Редактировать» показываются белые вершины. Их можно таскать ЛКМ.

## Безопасное обновление

1. Останови текущую версию:

```powershell
docker compose down
```

2. **Сохрани свою папку `backend/data`** из старого проекта — в ней база и все загруженные изображения:

```powershell
Copy-Item -Recurse -Force .\backend\data ..\atlas-data-backup
```

3. Распакуй v0.4 в отдельную папку.
4. Скопируй сохранённую `backend/data` поверх `backend/data` в новой папке.
5. Скопируй `.env` из старой папки, если он уже содержит твой `ADMIN_PASSWORD` и `AUTH_SECRET`.
6. Запусти:

```powershell
docker compose up --build
```

При первом старте v0.4 автоматически добавит в SQLite поле пропорций изображения (`aspect_ratio`). Старые данные и загрузки сохранятся.
